Python notes

Интерпретатор python:
- корневой;
- отдельный.
===============================================================================

Виртуальное окружение (virtual environment) - это изоляция python проекта от
корневых интерпретатора и зависимостей путем помещения в проектном каталоге
отдельного интерпретатора python с необходимыми версиями сторонних библиотек.
При коммите передается только специальный файл requirements.txt с перечнем
зависимостей и их версий.

Создание виртуального окружения:
$ python3 -m venv venv

-m                        - запуск модуля библиотеки как скрипта
venv (последний параметр) - имя виртуального окружения

Активация виртуального окружения

Linux:
$ source venv/bin/activate

Win:
venv\Scripts\activate

Деактивация:
$ deactivate

При активном виртуальном окружении все сторонние библиотеки будут устанавливаться в текущий проектный каталог.

Просмотр установленных зависимостей
$ pip freeze

Вывод в файл:
$ pip freeze > requirements.txt

Файл requirements.txt находится под управлением гита!

Баг в Ubuntu: В файле requirements.txt самим удалить следующую возможную строку:
pkg_resources==0.0.0

При передаче, после активации виртуального окружения:
$ pip install -r requirements.txt

После активации виртуального окружения хорошей практикой является обновление модулей pip и setuptools:
$ pip install -U pip setuptools

-U - upgrade

Установка библиотеки:
$ pip install requests lxml

Удаление библиотеки:
$ pip uninstall lxm -y

-y - опциональный параметр подтверждающий удаление без промта

Удаление виртуального окружения:
$ rm -rfv venv
